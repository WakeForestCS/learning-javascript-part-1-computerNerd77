What is Event-driven programming

This is part of javascript where if something happens, then you can execute something else. Events can be from the user such as a click on the page or from background processes like if the server gets a package and wants to tell the user the package is here.

Functions are first class citizens/objects

In Javascript, something really cool that happens is functions can be used like variables. So, just like any other variable, can be passed as a parameter to another function and can return the function from a function. One way this can be done, as stated later, is through callback functions.

Scope

Scope is actually common in computer science, just now with Javascript, there is a specific word for it. Scope relates to the accessibiltiy of variables. Just like in C, declaring a variable in a function only makes it valid for that function. This is true for Javascript, and using this example, would be considered a local scope: only access in the function. The other type of scope is global scope, declared outside of a function so that anyone can access it.

Callback Functions

This relates back to using functions as first class objects. Because of this, it opens up new doors. One of my callback function for this page relates to counting how many times you have clicked the do something button.

You can check this number on the Javascript console through typing window.count.

Basics of web application via Express

Thus far, we've only talked about client side code, but this method resolves talking to the server. This I will focus on the GET request; this allows the user to grab the client side code on their browser. This is what the internet is for most users: a series of GET request to grab code and display that on their browser.

It is possible to grab this code from a personal computer if the server is running but usually due to the conflict of different IP addresses, a simple GET request does not allow for that. This is most likely due to security reasons, which is great in general but prevents the ability to show off.

This is my .js code

const express = require('express');

const app = express();


app.listen(3000, function(){

console.log('listening on 3000')

})


app.get('/', (req, res)=>{

res.sendFile(__dirname + '/myPage.html')

})
